76/*
77 * New world logging!
78 * The brief summary:
79 * Logging used to be a simple affair, a FILE * handled by a nonblocking logging class inheriting from EventHandler, that was inserted
80 * into the socket engine, and wrote lines. If nofork was on, it was printf()'d.
81 *
82 * We decided to horribly overcomplicate matters, and create vastly customisable logging. LogManager and LogStream form the visible basis
83 * of the new interface. Basically, a LogStream can be inherited to do different things with logging output. We inherit from it once in core
84 * to create a FileLogStream, that writes to a file, for example. Different LogStreams can hook different types of log messages, and different
85 * levels of output too, for extreme customisation. Multiple LogStreams can hook the same message/levels of output, meaning that e.g. output
86 * can go to a channel as well as a file.
87 *
88 * HOW THIS WORKS
89 * LogManager handles all instances of LogStreams, classes derived from LogStream are instantiated and passed to it.
90 */
